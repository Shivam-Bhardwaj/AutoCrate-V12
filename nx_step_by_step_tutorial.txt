STEP 1: PREPARING THE EXPRESSION FILE
========================================================================

BEFORE YOU START NX:
Create or verify your expression file has the following format:

```
// KLIMP 1 - STANDARD ORIENTATION
KL_1_SUPPRESS = 1
[Inch]KL_1_X = 10.000
[Inch]KL_1_Y = 10.000
[Inch]KL_1_Z = 10.000
KL_1_X_DIR_X = 1.000000
KL_1_X_DIR_Y = 0.000000
KL_1_X_DIR_Z = 0.000000
KL_1_Y_DIR_X = 0.000000
KL_1_Y_DIR_Y = 1.000000
KL_1_Y_DIR_Z = 0.000000
KL_1_Z_DIR_X = 0.000000
KL_1_Z_DIR_Y = 0.000000
KL_1_Z_DIR_Z = 1.000000

// KLIMP 2 - FACING INWARD
KL_2_SUPPRESS = 1
[Inch]KL_2_X = 20.000
[Inch]KL_2_Y = 10.000
[Inch]KL_2_Z = 10.000
KL_2_X_DIR_X = -1.000000
KL_2_X_DIR_Y = 0.000000
KL_2_X_DIR_Z = 0.000000
KL_2_Y_DIR_X = 0.000000
KL_2_Y_DIR_Y = 1.000000
KL_2_Y_DIR_Z = 0.000000
KL_2_Z_DIR_X = 0.000000
KL_2_Z_DIR_Y = 0.000000
KL_2_Z_DIR_Z = -1.000000

// KLIMP 3 - SUPPRESSED (HIDDEN)
KL_3_SUPPRESS = 0
[Inch]KL_3_X = 0.000
[Inch]KL_3_Y = 0.000
[Inch]KL_3_Z = 0.000
KL_3_X_DIR_X = 1.000000
KL_3_X_DIR_Y = 0.000000
KL_3_X_DIR_Z = 0.000000
KL_3_Y_DIR_X = 0.000000
KL_3_Y_DIR_Y = 1.000000
KL_3_Y_DIR_Z = 0.000000
KL_3_Z_DIR_X = 0.000000
KL_3_Z_DIR_Y = 0.000000
KL_3_Z_DIR_Z = 1.000000

// ... Continue for all 30 klimps
```

CRITICAL REQUIREMENTS:
□ All direction vectors must have unit length (1.0)
□ X, Y, Z directions must be orthogonal (perpendicular)
□ Use consistent naming: KL_XX_VARIABLENAME
□ Include [Inch] prefix for position variables only
□ Save file with .exp extension

========================================================================
STEP 2: SETTING UP THE NX ASSEMBLY FILE
========================================================================

STEP 2.1: CREATE NEW ASSEMBLY
-----------------------------
1. Launch Siemens NX
2. Click "File" → "New"
3. Select "Assembly" template
4. Choose appropriate units (English - inches)
5. Enter filename: "Crate_Assembly.prt"
6. Click "OK"

STEP 2.2: SET UP ASSEMBLY ENVIRONMENT
------------------------------------
1. Verify you're in "Assembly" application
   - Look for assembly toolbar at top
   - Should see "Component" menu options
2. Set up your workspace:
   - Graphics → View → Isometric
   - View → Fit

STEP 2.3: SAVE ASSEMBLY FILE
---------------------------
1. File → Save As
2. Choose location near your expression file
3. Filename: "Crate_Assembly_with_Klimps.prt"
4. Click "Save"

========================================================================
STEP 3: IMPORTING EXPRESSION VARIABLES
========================================================================

STEP 3.1: OPEN EXPRESSION EDITOR
--------------------------------
1. Click "Tools" menu
2. Select "Expression"
3. Expression dialog box opens

STEP 3.2: IMPORT EXPRESSION FILE
-------------------------------
1. In Expression dialog, click "Import" button
2. Navigate to your .exp file location
3. Select your klimp expression file
4. Click "Open"

STEP 3.3: VERIFY IMPORT SUCCESS
------------------------------
1. Scroll through expression list
2. Verify you see variables like:
   - KL_1_SUPPRESS
   - KL_1_X
   - KL_1_X_DIR_X
   - etc.
3. Check for any error messages
4. Count total expressions (should be ~390 for 30 klimps)

STEP 3.4: TEST EXPRESSIONS
--------------------------
1. Find "KL_1_X" in the list
2. Double-click to edit
3. Change value to 15.000
4. Click "Apply"
5. Change back to 10.000
6. Click "Apply"
7. This confirms expressions are working
8. Click "Close" to exit Expression editor

STEP 4: CREATING PARAMETRIC COORDINATE SYSTEMS
========================================================================

IMPORTANT: We'll create one coordinate system for each klimp position.
Start with klimp 1, then repeat for all 30 klimps.

STEP 4.1: CREATE FIRST COORDINATE SYSTEM (KLIMP 1)
-------------------------------------------------
1. Click "Insert" menu
2. Select "Datum/Point" → "Coordinate System"
3. Coordinate System dialog opens

STEP 4.2: CONFIGURE COORDINATE SYSTEM ORIGIN
-------------------------------------------
1. In "Origin" section:
   - X field: Type "KL_1_X" (without quotes)
   - Y field: Type "KL_1_Y"
   - Z field: Type "KL_1_Z"
2. These fields should turn blue (indicating expression link)

STEP 4.3: CONFIGURE X-AXIS DIRECTION
-----------------------------------
1. Click "Type" dropdown → Select "By Vectors"
2. In "X Axis" section:
   - X Component: Type "KL_1_X_DIR_X"
   - Y Component: Type "KL_1_X_DIR_Y"
   - Z Component: Type "KL_1_X_DIR_Z"
3. All fields should turn blue

STEP 4.4: CONFIGURE Y-AXIS DIRECTION
-----------------------------------
1. In "Y Axis" section:
   - X Component: Type "KL_1_Y_DIR_X"
   - Y Component: Type "KL_1_Y_DIR_Y"
   - Z Component: Type "KL_1_Y_DIR_Z"
2. All fields should turn blue

STEP 4.5: FINALIZE COORDINATE SYSTEM
-----------------------------------
1. Click "OK" to create coordinate system
2. New coordinate system appears at position (10, 10, 10)
3. Note the coordinate system name (e.g., "CSYS(1)")

STEP 4.6: RENAME COORDINATE SYSTEM
---------------------------------
1. In Feature Navigator (left panel):
   - Find the new coordinate system
   - Right-click on it
   - Select "Rename"
   - Enter name: "KLIMP_01_CSYS"
   - Press Enter

STEP 4.7: VERIFY COORDINATE SYSTEM
---------------------------------
1. The coordinate system should:
   - Be located at (10, 10, 10)
   - Have X-axis pointing in +X direction (red arrow)
   - Have Y-axis pointing in +Y direction (green arrow)
   - Have Z-axis pointing in +Z direction (blue arrow)

STEP 4.8: CREATE REMAINING COORDINATE SYSTEMS
--------------------------------------------
Repeat steps 4.1-4.7 for each klimp, changing only the variable names:

For Klimp 2:
- Origin: KL_2_X, KL_2_Y, KL_2_Z
- X-Axis: KL_2_X_DIR_X, KL_2_X_DIR_Y, KL_2_X_DIR_Z
- Y-Axis: KL_2_Y_DIR_X, KL_2_Y_DIR_Y, KL_2_Y_DIR_Z
- Name: "KLIMP_02_CSYS"

For Klimp 3:
- Origin: KL_3_X, KL_3_Y, KL_3_Z
- X-Axis: KL_3_X_DIR_X, KL_3_X_DIR_Y, KL_3_X_DIR_Z
- Y-Axis: KL_3_Y_DIR_X, KL_3_Y_DIR_Y, KL_3_Y_DIR_Z
- Name: "KLIMP_03_CSYS"

... Continue for all 30 klimps

SHORTCUT: COPY AND MODIFY
------------------------
After creating the first coordinate system:
1. Right-click "KLIMP_01_CSYS" in Feature Navigator
2. Select "Edit Parameters"
3. Copy all the expressions
4. Create new coordinate system
5. Paste and modify the klimp number

========================================================================
STEP 5: INSERTING AND CONSTRAINING KLIMP COMPONENTS
========================================================================

STEP 5.1: ADD FIRST KLIMP COMPONENT
----------------------------------
1. Click "Assemblies" → "Add Component"
2. Navigate to your klimp part file
3. Select the klimp .prt file
4. Click "Open"
5. Position klimp roughly near KLIMP_01_CSYS
6. Click to place component

STEP 5.2: CONSTRAIN KLIMP TO COORDINATE SYSTEM
---------------------------------------------
1. Click "Assemblies" → "Assembly Constraints"
2. Assembly Constraints dialog opens

STEP 5.3: CREATE MATE CONSTRAINT
-------------------------------
1. Constraint Type: Select "Mate"
2. First Selection:
   - Click on klimp component
   - Select klimp's local coordinate system origin
3. Second Selection:
   - Click on KLIMP_01_CSYS origin point
4. Click "OK" to apply constraint

STEP 5.4: CREATE ALIGN CONSTRAINTS
---------------------------------
You need to align the axes. Create three alignment constraints:

ALIGN X-AXES:
1. Constraint Type: "Align"
2. First Selection: Klimp's local X-axis
3. Second Selection: KLIMP_01_CSYS X-axis
4. Click "OK"

ALIGN Y-AXES:
1. Constraint Type: "Align"
2. First Selection: Klimp's local Y-axis
3. Second Selection: KLIMP_01_CSYS Y-axis
4. Click "OK"

ALIGN Z-AXES:
1. Constraint Type: "Align"
2. First Selection: Klimp's local Z-axis
3. Second Selection: KLIMP_01_CSYS Z-axis
4. Click "OK"

STEP 5.5: VERIFY KLIMP POSITIONING
---------------------------------
1. Klimp should now be exactly positioned at (10, 10, 10)
2. Klimp should be oriented according to direction vectors
3. No degrees of freedom should remain

STEP 5.6: TEST PARAMETRIC BEHAVIOR
---------------------------------
1. Tools → Expression
2. Find "KL_1_X"
3. Change value to 15.000
4. Click "Apply"
5. Klimp should move to new X position
6. Change back to 10.000
7. Click "Apply"
8. Klimp returns to original position
9. Close Expression dialog

STEP 5.7: REPEAT FOR ALL KLIMPS
------------------------------
Repeat steps 5.1-5.6 for each klimp:
- Add component
- Constrain to corresponding coordinate system
- Test parametric behavior

========================================================================
STEP 6: SETTING UP SUPPRESS LOGIC
========================================================================

STEP 6.1: SET UP CONDITIONAL SUPPRESSION
---------------------------------------
1. In Feature Navigator, find first klimp component
2. Right-click on klimp component
3. Select "Properties"
4. Component Properties dialog opens

STEP 6.2: CONFIGURE SUPPRESSION CONDITION
----------------------------------------
1. Click "Conditional" tab
2. Check "Suppress using expression"
3. In expression field, enter: "KL_1_SUPPRESS == 0"
   - Note: This suppresses when KL_1_SUPPRESS equals 0
   - When KL_1_SUPPRESS = 1, klimp shows
   - When KL_1_SUPPRESS = 0, klimp hides
4. Click "OK"

STEP 6.3: TEST SUPPRESS FUNCTIONALITY
------------------------------------
1. Tools → Expression
2. Find "KL_1_SUPPRESS"
3. Change value to 0
4. Click "Apply"
5. Klimp should disappear
6. Change value back to 1
7. Click "Apply"
8. Klimp should reappear
9. Close Expression dialog

STEP 6.4: APPLY TO ALL KLIMPS
----------------------------
Repeat steps 6.1-6.3 for each klimp component:
- KL_2_SUPPRESS == 0
- KL_3_SUPPRESS == 0
- ... etc.

STEP 8: TROUBLESHOOTING COMMON ISSUES
========================================================================

ISSUE 1: "INVALID COORDINATE SYSTEM"
----------------------------------
SYMPTOMS: NX won't create coordinate system
CAUSE: Direction vectors not orthogonal or not unit length

SOLUTION:
1. Check expression values in Expression editor
2. Verify each direction vector has length 1.0:
   - √(X² + Y² + Z²) = 1.0
3. Verify orthogonality (dot product = 0):
   - X·Y = 0, Y·Z = 0, Z·X = 0
4. Use expression to calculate and verify:
   - Create expression: "MAGNITUDE_X = sqrt(KL_1_X_DIR_X^2 + KL_1_X_DIR_Y^2 + KL_1_X_DIR_Z^2)"
   - Should equal 1.0

ISSUE 2: "KLIMP POINTS WRONG DIRECTION"
-------------------------------------
SYMPTOMS: Klimp orientation doesn't match expected direction
CAUSE: Confusion about axis definitions

SOLUTION:
1. Verify klimp part local coordinate system
2. Ensure klimp "front" aligns with local X-axis
3. Ensure klimp "top" aligns with local Y-axis
4. Use simple test orientations first:
   - X_DIR = (1,0,0), Y_DIR = (0,1,0) for standard orientation

ISSUE 3: "EXPRESSIONS DON'T LINK"
-------------------------------
SYMPTOMS: Expression fields stay black instead of blue
CAUSE: Typo in expression name or expression doesn't exist

SOLUTION:
1. Verify exact spelling: "KL_1_X" not "KL_1_x"
2. Check expression exists in Expression editor
3. Use copy/paste to avoid typos
4. Verify no extra spaces in expression names

ISSUE 4: "KLIMP DOESN'T MOVE WITH EXPRESSIONS"
--------------------------------------------
SYMPTOMS: Changing expressions doesn't update klimp position
CAUSE: Constraints not properly linked to coordinate system

SOLUTION:
1. Edit klimp constraints
2. Verify all constraints reference the parametric coordinate system
3. Check for over-constrained condition
4. Ensure coordinate system is expression-driven

ISSUE 5: "SUPPRESS FLAG DOESN'T WORK"
-----------------------------------
SYMPTOMS: Changing suppress flag doesn't hide/show klimp
CAUSE: Wrong conditional expression or syntax error

SOLUTION:
1. Right-click klimp → Properties → Conditional
2. Verify expression: "KL_XX_SUPPRESS == 0" (note double equals)
3. Test expression in Expression editor first
4. Check for typos in variable name

ISSUE 6: "COORDINATE SYSTEM FLIPS UNEXPECTEDLY"
---------------------------------------------
SYMPTOMS: Coordinate system Z-axis points wrong direction
CAUSE: Non-right-handed coordinate system

SOLUTION:
1. Verify right-hand rule: Z = X × Y
2. If Z points wrong way, negate all Z-direction components
3. Or swap Y and Z directions
4. Test with simple orientations first
