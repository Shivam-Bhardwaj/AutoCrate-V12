==================================================
1. NX COORDINATE SYSTEM FEATURES
==================================================

NX provides several ways to create coordinate systems using vectors:

METHOD 1: COORDINATE SYSTEM BY VECTORS (RECOMMENDED)
---------------------------------------------------
- Create coordinate system using three orthogonal vectors
- Most direct implementation of our direction vector approach
- Uses expressions to define vector directions
- Fully parametric and updateable

METHOD 2: COORDINATE SYSTEM BY POINTS
-------------------------------------
- Define coordinate system using three points
- Can be calculated from direction vectors
- Useful when you have specific target points

METHOD 3: COORDINATE SYSTEM BY ANGLES (AVOID)
---------------------------------------------
- Uses Euler angles (our old problematic method)
- Subject to gimbal lock
- We're moving away from this approach

==================================================
2. EXPRESSION-DRIVEN COORDINATE SYSTEMS
==================================================

NX COORDINATE SYSTEM DIALOG INPUTS:
----------------------------------
When creating a coordinate system in NX, you can specify:

ORIGIN:
- X coordinate
- Y coordinate  
- Z coordinate

X-AXIS DIRECTION:
- X component
- Y component
- Z component

Y-AXIS DIRECTION:
- X component
- Y component
- Z component

(Z-axis is automatically calculated as X × Y)

EXPRESSION VARIABLE MAPPING:
---------------------------
Our klimp variables map directly to NX inputs:

Origin Point:
- X Coordinate = KL_XX_X
- Y Coordinate = KL_XX_Y
- Z Coordinate = KL_XX_Z

X-Axis Direction:
- X Component = KL_XX_X_DIR_X
- Y Component = KL_XX_X_DIR_Y
- Z Component = KL_XX_X_DIR_Z

Y-Axis Direction:
- X Component = KL_XX_Y_DIR_X
- Y Component = KL_XX_Y_DIR_Y
- Z Component = KL_XX_Y_DIR_Z

==================================================
3. STEP-BY-STEP IMPLEMENTATION
==================================================

STEP 1: PREPARE EXPRESSION FILE
------------------------------
Update your .exp file to include direction vector variables:

```
// KLIMP 1 POSITION AND ORIENTATION
KL_1_SUPPRESS = 1
[Inch]KL_1_X = 25.000
[Inch]KL_1_Y = 30.000
[Inch]KL_1_Z = 15.000

// KLIMP 1 X-AXIS DIRECTION (WHERE KLIMP POINTS)
KL_1_X_DIR_X = 0.866025
KL_1_X_DIR_Y = 0.000000
KL_1_X_DIR_Z = 0.500000

// KLIMP 1 Y-AXIS DIRECTION (KLIMP UP DIRECTION)
KL_1_Y_DIR_X = 0.000000
KL_1_Y_DIR_Y = 1.000000
KL_1_Y_DIR_Z = 0.000000

// KLIMP 1 Z-AXIS DIRECTION (KLIMP SIDE DIRECTION - CALCULATED)
KL_1_Z_DIR_X = -0.500000
KL_1_Z_DIR_Y = 0.000000
KL_1_Z_DIR_Z = 0.866025
```

STEP 2: IMPORT EXPRESSIONS INTO NX
----------------------------------
1. Open your NX part file
2. Go to Tools → Expression
3. Import your .exp file
4. Verify all variables are loaded correctly

STEP 3: CREATE PARAMETRIC COORDINATE SYSTEM
------------------------------------------
1. Go to Insert → Datum/Point → Coordinate System
2. Choose "By Vectors" method
3. Set up as follows:

ORIGIN POINT:
- X: Enter expression "KL_1_X"
- Y: Enter expression "KL_1_Y"  
- Z: Enter expression "KL_1_Z"

X-AXIS DIRECTION:
- X Component: Enter expression "KL_1_X_DIR_X"
- Y Component: Enter expression "KL_1_X_DIR_Y"
- Z Component: Enter expression "KL_1_X_DIR_Z"

Y-AXIS DIRECTION:
- X Component: Enter expression "KL_1_Y_DIR_X"
- Y Component: Enter expression "KL_1_Y_DIR_Y"
- Z Component: Enter expression "KL_1_Y_DIR_Z"

4. Click OK to create coordinate system

STEP 4: POSITION KLIMP PART USING COORDINATE SYSTEM
--------------------------------------------------
1. Insert your klimp component
2. Use Assembly Constraints → Component Position
3. Set constraint type to "Fix" or "Mate"
4. Align klimp's local coordinate system with the parametric coordinate system you created
5. The klimp will now follow the expressions automatically

STEP 5: CREATE SUPPRESS/UNSUPPRESS LOGIC
---------------------------------------
1. Right-click on klimp component
2. Go to Properties → Conditional
3. Set condition: "KL_1_SUPPRESS == 1"
4. This will show/hide the klimp based on the SUPPRESS flag

==================================================
4. EXPRESSION FILE FORMAT
==================================================

COMPLETE KLIMP DEFINITION (RECOMMENDED FORMAT):
----------------------------------------------
```
// === KLIMP 1 (TOP PANEL) ===
KL_1_SUPPRESS = 1                    // 0=hide, 1=show

// Position (origin of local coordinate system)
[Inch]KL_1_X = 25.000               // X coordinate in global system
[Inch]KL_1_Y = 30.000               // Y coordinate in global system  
[Inch]KL_1_Z = 15.000               // Z coordinate in global system

// Local X-axis direction (klimp "front" direction)
KL_1_X_DIR_X = 0.866025             // X component of klimp X-axis
KL_1_X_DIR_Y = 0.000000             // Y component of klimp X-axis
KL_1_X_DIR_Z = 0.500000             // Z component of klimp X-axis

// Local Y-axis direction (klimp "up" direction)  
KL_1_Y_DIR_X = 0.000000             // X component of klimp Y-axis
KL_1_Y_DIR_Y = 1.000000             // Y component of klimp Y-axis
KL_1_Y_DIR_Z = 0.000000             // Z component of klimp Y-axis

// Local Z-axis direction (klimp "side" direction - computed)
KL_1_Z_DIR_X = -0.500000            // X component of klimp Z-axis
KL_1_Z_DIR_Y = 0.000000             // Y component of klimp Z-axis
KL_1_Z_DIR_Z = 0.866025             // Z component of klimp Z-axis

// Legacy compatibility (optional - for transition period)
KL_1_RX = 0.000                     // Legacy X rotation (deprecated)
KL_1_RY = 30.000                    // Legacy Y rotation (deprecated)
KL_1_RZ = 0.000                     // Legacy Z rotation (deprecated)
KL_1_ROTATE = 30                    // Legacy single rotation (deprecated)
```

==================================================
5. NX MODELING WORKFLOW
==================================================

WORKFLOW FOR KLIMP MASTER PART:
------------------------------
1. Create base klimp geometry
2. Establish local coordinate system at klimp origin
3. Model klimp relative to this local coordinate system
4. Ensure klimp "points" along local X-axis
5. Ensure klimp "up" is along local Y-axis

WORKFLOW FOR ASSEMBLY:
---------------------
1. Create parametric coordinate systems for each klimp position
2. Insert klimp instances
3. Constrain each klimp to its coordinate system
4. Set up conditional visibility based on SUPPRESS flags
5. Test by updating expressions

COORDINATE SYSTEM NAMING CONVENTION:
----------------------------------
- Name coordinate systems: "KLIMP_01_CSYS", "KLIMP_02_CSYS", etc.
- This makes it easy to identify which coordinate system belongs to which klimp
- Helps with debugging and maintenance

ASSEMBLY CONSTRAINT STRATEGY:
----------------------------
For each klimp:
1. Use "Mate" constraint between klimp local CS and parametric CS
2. Ensure proper alignment (klimp X-axis aligns with CS X-axis, etc.)
3. Lock all degrees of freedom
4. Klimp position/orientation now follows expressions automatically

==================================================
6. TESTING AND VALIDATION
==================================================

VALIDATION CHECKLIST:
--------------------
□ All direction vectors are unit length (magnitude = 1)
□ X, Y, Z directions are orthogonal (perpendicular)
□ Right-handed coordinate system (Z = X × Y)
□ Klimp points in expected direction
□ Klimp orientation matches design intent
□ SUPPRESS flag works correctly
□ Expression updates propagate to model

TESTING PROCEDURE:
-----------------
1. Create test expressions with known orientations:
   - Identity matrix (no rotation)
   - 90° rotation about each axis
   - 180° rotation (face opposite direction)
   - 45° tilt toward center

2. Import expressions and verify visual results

3. Test parametric updates:
   - Change position → klimp should move
   - Change direction → klimp should reorient
   - Change suppress → klimp should hide/show

COMMON TEST CASES:
-----------------
```
// Test 1: Standard orientation
KL_1_X_DIR = (1, 0, 0)  // Points right
KL_1_Y_DIR = (0, 1, 0)  // Points up
KL_1_Z_DIR = (0, 0, 1)  // Points forward

// Test 2: 90° rotation about Z
KL_2_X_DIR = (0, 1, 0)  // Points where Y used to point
KL_2_Y_DIR = (-1, 0, 0) // Points where -X used to point
KL_2_Z_DIR = (0, 0, 1)  // Still points forward

// Test 3: Face opposite direction
KL_3_X_DIR = (-1, 0, 0) // Points left
KL_3_Y_DIR = (0, 1, 0)  // Still points up
KL_3_Z_DIR = (0, 0, -1) // Points backward
```

7. COMMON ISSUES AND SOLUTIONS
==================================================

ISSUE 1: "INVALID COORDINATE SYSTEM"
-----------------------------------
Cause: Direction vectors are not orthogonal or not unit length
Solution: 
- Check vector calculations
- Verify dot products are zero
- Verify vector magnitudes are 1.0

ISSUE 2: "KLIMP POINTS WRONG DIRECTION"
--------------------------------------
Cause: Confusion about which axis is which
Solution:
- Establish clear convention (X = front, Y = up, Z = side)
- Test with simple orientations first
- Use right-hand rule to verify

ISSUE 3: "KLIMP FLIPS UNEXPECTEDLY"
----------------------------------
Cause: Z-axis calculation gives unexpected result
Solution:
- Always compute Z = X × Y (cross product)
- Check right-handed system convention
- May need to negate Z direction

ISSUE 4: "EXPRESSIONS DON'T UPDATE MODEL"
----------------------------------------
Cause: Coordinate system not properly linked to expressions
Solution:
- Verify expression names match exactly
- Check for typos in expression references
- Use Expression Manager to verify links

ISSUE 5: "SUPPRESS FLAG DOESN'T WORK"
------------------------------------
Cause: Conditional suppression not set up correctly
Solution:
- Right-click component → Properties → Conditional
- Set condition exactly: "KL_XX_SUPPRESS == 1"
- Use "==" not "=" for comparison

ADVANCED TIPS:
=============

TIP 1: USE REFERENCE COORDINATE SYSTEMS
- Create one master coordinate system for testing
- Copy/modify for each klimp position
- Easier than creating each from scratch

TIP 2: BATCH CREATE COORDINATE SYSTEMS
- Record journal for creating one coordinate system
- Modify journal to loop through all klimps
- Automates creation of 30 coordinate systems

TIP 3: EXPRESSION-DRIVEN COLORS
- Use expressions to control klimp colors
- Different colors for different orientations
- Helps visualize and debug orientations

TIP 4: CREATE DIRECTION VECTOR VISUALIZATION
- Create small arrow parts showing X, Y, Z directions
- Position arrows using same expressions
- Makes it easy to verify orientations visually

SAMPLE NX JOURNAL CODE:
----------------------
```
' Simple NX Journal to create klimp coordinate system
Option Strict Off
Imports System
Imports NXOpen

Module CreateKlimpCoordinateSystem

Sub Main()
    Dim theSession As Session = Session.GetSession()
    Dim workPart As Part = theSession.Parts.Work
    
    ' Create coordinate system
    Dim csysBuilder As Features.CoordinateSystemBuilder
    csysBuilder = workPart.Features.CreateCoordinateSystemBuilder(Nothing)
    
    ' Set origin
    csysBuilder.Origin.SetCoordinates("KL_1_X", "KL_1_Y", "KL_1_Z")
    
    ' Set X-axis direction
    csysBuilder.XAxisDirection.SetVector("KL_1_X_DIR_X", "KL_1_X_DIR_Y", "KL_1_X_DIR_Z")
    
    ' Set Y-axis direction  
    csysBuilder.YAxisDirection.SetVector("KL_1_Y_DIR_X", "KL_1_Y_DIR_Y", "KL_1_Y_DIR_Z")
    
    ' Commit the feature
    Dim csysFeature As Features.CoordinateSystem
    csysFeature = csysBuilder.Commit()
    
    csysBuilder.Destroy()
End Sub

End Module
```

SUMMARY:
=======
1. Use NX "Coordinate System by Vectors" feature
2. Link coordinate system to expression variables
3. Position klimps using coordinate system constraints
4. Test with simple orientations first
5. Verify orthogonality and unit length of vectors

This approach gives you full parametric control over klimp positions and orientations without gimbal lock limitations!
